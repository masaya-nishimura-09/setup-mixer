# Amazon Product Advertising API 調査報告書

**調査日**: 2024 年  
**調査目的**: オフィスグッズセット作成サービスにおける商品データ取得方法の現実性評価

---

## 1. 調査サマリー

### 1.1 結論

**Amazon Product Advertising API（PA-API）の利用は、MVP 段階では現実的ではない可能性が高い**

主な理由:

- アソシエイトプログラムへの参加が必須
- 継続的な売上実績が必要（30 日間で売上がないとアクセス停止）
- 初期段階でのリクエスト制限が厳しい
- 売上実績に応じて制限が緩和される仕組み

### 1.2 推奨アプローチ

1. **MVP 段階**: サンプルデータ（手動または代替 API）でプロトタイプを作成
2. **サービス公開後**: アソシエイトプログラムに参加し、売上実績を積む
3. **Phase 2 以降**: 十分な実績ができた時点で PA-API の利用を検討

---

## 2. Amazon Product Advertising API（PA-API）詳細

### 2.1 基本情報

- **正式名称**: Amazon Product Advertising API 5.0（PA-API 5.0）
- **提供元**: Amazon Web Services
- **用途**: Amazon 商品データベースへのアクセス、商品情報・価格・レビューの取得
- **料金**: API 利用自体は無料（ただし、アソシエイトプログラムの条件を満たす必要あり）

### 2.2 利用条件

#### 2.2.1 必須要件

1. **Amazon アソシエイト・プログラムへの参加**

   - アソシエイト ID の取得が必要
   - ウェブサイトまたはアプリケーションの公開が前提条件となる可能性

2. **AWS アカウント**
   - AWS アクセスキーとシークレットキーの取得が必要

#### 2.2.2 利用制限

**初期制限（最初の 30 日間）:**

- **リクエスト速度**: 1 秒あたり 1 リクエスト（1 TPS）
- **1 日の上限**: 最大 8,640 リクエスト（8,640 TPD）
- **計算**: 8,640 リクエスト ÷ 86,400 秒 = 1 リクエスト/秒

**制限の緩和条件:**

- **売上に基づく増加**:
  - 5 セント（約 7.5 円）の売上ごとに 1 TPD（1 日あたりのリクエスト数）が追加
  - 4,320 ドル（約 65 万円）の売上ごとに 1 TPS（秒あたりのリクエスト数）が追加
  - 最大 10 TPS まで増加可能

**例:**

- 1 日 100 リクエストの商品検索を実行する場合: 約 1.2 秒に 1 回のリクエスト
- 14 カテゴリ × 各 20 商品 = 280 商品を取得する場合: 約 4.7 分かかる（初期制限下）

#### 2.2.3 継続的な利用要件

**重要な制約:**

- **過去 30 日間で適格な紹介売上が発生していない場合、PA-API へのアクセスが失われる可能性がある**
- これは「鶏と卵」の問題を引き起こす:
  - API を使うには売上が必要
  - 売上を得るには商品データが必要
  - 商品データを得るには API が必要

### 2.3 取得可能なデータ

PA-API で取得可能な情報:

- 商品名、説明
- 価格情報
- 商品画像
- レビュー情報（平均評価、レビュー件数）
- 在庫状況
- 商品カテゴリ
- ASIN（Amazon 商品識別子）
- アフィリエイトリンクの生成

### 2.4 利用規約・制限事項

1. **データのキャッシュ・保存**

   - 取得したデータの保存方法には制限がある
   - Amazon のポリシーに従う必要がある

2. **価格表示の制限**

   - 一部商品では、メーカーの最低広告価格（MAP）により、特定条件下でのみ価格が表示される

3. **利用目的の制限**
   - 主要な e コマース事業（オンラインショッピング事業やオンライン比較ショッピングサービスなど）を運営している企業は利用不可の可能性

## 3. 代替手段の検討

### 3.1 楽天市場 API

#### 3.1.1 基本情報

- **提供元**: 楽天グループ
- **API 名**: 楽天市場 API（楽天ウェブサービス）
- **料金**: **無料**（商用利用も可能）
- **申請**: 楽天ウェブサービスに登録が必要（比較的簡単）

#### 3.1.2 利用条件

- 楽天ウェブサービスへの登録（無料）
- アプリケーション ID の取得
- 利用規約の遵守

#### 3.1.3 取得可能なデータ

- 商品検索結果
- 商品詳細情報
- 商品画像
- 価格情報
- レビュー情報（一部）
- カテゴリ情報
- アフィリエイトリンクの生成（楽天アフィリエイトプログラム経由）

#### 3.1.4 メリット・デメリット

**メリット:**

- ✅ 無料で利用可能
- ✅ 申請が比較的簡単
- ✅ リクエスト制限が緩い（1 秒あたり 10 リクエスト程度）
- ✅ 商用利用可能
- ✅ 日本語ドキュメントが充実

**デメリット:**

- ⚠️ Amazon 商品は取得できない
- ⚠️ 商品数が Amazon より少ない可能性
- ⚠️ アフィリエイト報酬率が異なる

#### 3.1.5 推奨度

**★★★★☆（高）**

MVP 段階では楽天 API を主軸とし、Amazon は後から追加する戦略が現実的。

### 3.2 バリューコマース API

#### 3.2.1 基本情報

- **提供元**: バリューコマース
- **料金**: 要確認（アフィリエイトプログラム経由）
- **特徴**: 複数の EC サイト（Amazon、楽天、Yahoo!ショッピングなど）の商品を統合

#### 3.2.2 推奨度

**★★★☆☆（中）**

複数サイトの商品を統合できるが、利用条件や料金体系の確認が必要。

### 3.3 スクレイピング

#### 3.3.1 基本情報

- **方法**: ウェブスクレイピング技術で商品ページから直接データを取得
- **技術**: Python（BeautifulSoup、Selenium）、Node.js（Puppeteer、Cheerio）など

#### 3.3.2 リスク

**⚠️ 高リスク - 推奨しない**

1. **利用規約違反の可能性**

   - Amazon の利用規約でスクレイピングが禁止されている可能性が高い
   - robots.txt でクローリングが制限されている

2. **法的リスク**

   - 著作権侵害の可能性
   - 不正アクセス禁止法に抵触する可能性

3. **技術的リスク**
   - IP アドレスのブロック
   - CAPTCHA の導入
   - HTML 構造の変更による取得失敗

#### 3.3.3 推奨度

**★☆☆☆☆（低）**

法的・倫理的なリスクが高く、長期的な運用には適さない。

### 3.4 手動データ入力

#### 3.4.1 方法

- 手動で商品情報を収集・入力
- CSV や JSON ファイルに保存

#### 3.4.2 メリット・デメリット

**メリット:**

- ✅ 法的リスクなし
- ✅ 完全なコントロール
- ✅ カスタムデータの追加が容易

**デメリット:**

- ❌ 時間がかかる
- ❌ データ更新が困難
- ❌ スケーラビリティが低い

#### 3.4.3 推奨度

**★★☆☆☆（低〜中）**

MVP のプロトタイプ用のサンプルデータ作成には有効だが、本番運用には不向き。

## 4. 推奨戦略

### 4.1 MVP 段階（Phase 1）

**推奨アプローチ: 楽天 API + サンプルデータ**

1. **楽天市場 API の利用**

   - 無料で利用可能
   - 申請が簡単
   - 十分な商品データが取得可能

2. **サンプルデータの作成**

   - 各カテゴリ 10-20 件の商品データを手動または楽天 API で取得
   - JSON ファイルに保存
   - プロトタイプの開発に使用

3. **Amazon 商品の扱い**
   - 手動で主要商品のみ追加（サンプルとして）
   - または、Amazon アソシエイトプログラムに参加し、手動でリンクを生成

### 4.2 Phase 2 以降

1. **Amazon アソシエイトプログラムへの参加**

   - サービスを公開
   - 楽天商品経由でユーザーを獲得
   - アソシエイトリンク経由で売上を発生させる

2. **売上実績の積み上げ**

   - 過去 30 日間で継続的な売上を確保
   - PA-API 利用の条件を満たす

3. **PA-API の導入**
   - 十分な実績ができた時点で申請
   - Amazon 商品データの自動取得を開始

### 4.3 ハイブリッド構成（推奨）

```
Phase 1:
├─ 楽天API: 主要データソース
├─ サンプルデータ: プロトタイプ用
└─ 手動追加: 人気商品のみ

Phase 2:
├─ 楽天API: 継続利用
├─ Amazon手動リンク: アソシエイトプログラム参加
└─ 売上実績の積み上げ

Phase 3:
├─ 楽天API: 継続利用
├─ Amazon PA-API: 実績ができた時点で導入
└─ その他ECサイト: 必要に応じて追加
```

## 5. 実装の現実性評価

### 5.1 Amazon PA-API のみの場合

**現実性: ★☆☆☆☆（低）**

- MVP 段階での利用は困難
- 売上実績が必要な「鶏と卵」の問題
- 初期制限が厳しく、大量の商品データ取得に時間がかかる

### 5.2 楽天 API + サンプルデータの場合

**現実性: ★★★★★（高）**

- 無料で利用可能
- 申請が簡単
- 十分な商品データが取得可能
- MVP の開発に最適

### 5.3 ハイブリッド構成の場合

**現実性: ★★★★☆（高）**

- MVP 段階では楽天 API で開始
- 段階的に Amazon を追加
- リスクを分散できる

---

## 6. 結論と推奨事項

### 6.1 結論

**プロジェクトを進めることは可能です**

ただし、以下の戦略を推奨します:

1. **MVP 段階**: 楽天市場 API を主軸とする
2. **サンプルデータ**: 楽天 API で取得したデータを JSON ファイルに保存
3. **Amazon**: Phase 2 以降で段階的に追加

### 6.2 次のステップ

1. **楽天ウェブサービスへの登録**

   - https://webservice.rakuten.co.jp/ にアクセス
   - アプリケーション ID を取得

2. **サンプルデータの取得**

   - 楽天 API で各カテゴリの商品を取得
   - JSON ファイルに保存

3. **プロトタイプの開発**

   - Next.js でプロトタイプを作成
   - 楽天商品データで動作確認

4. **Amazon アソシエイトプログラムの準備**
   - サービス公開後に参加を検討
   - 売上実績を積み上げる

### 6.3 リスク評価

| 項目                 | リスクレベル | 対策                   |
| -------------------- | ------------ | ---------------------- |
| 楽天 API の利用      | 低           | 利用規約を確認し、遵守 |
| Amazon PA-API の利用 | 中           | Phase 2 以降で検討     |
| データ取得の継続性   | 低           | 複数データソースを確保 |
| 法的リスク           | 低           | 公式 API のみを使用    |

## 8. 楽天 API の詳細情報

### 8.1 リクエスト制限

**公式情報:**

- **1 秒あたりのリクエスト数**: 通常 10 リクエスト/秒程度（公式ドキュメントで確認推奨）
- **1 日の上限**: 明確な制限は公開されていないが、通常の利用では問題なし
- **制限の緩さ**: Amazon PA-API と比較して、制限が非常に緩い

**実用的な観点:**

- 14 カテゴリ × 各 20 商品 = 280 商品を取得する場合
- 楽天 API: 約 28 秒（10 リクエスト/秒の場合）
- Amazon PA-API: 約 4.7 分（1 リクエスト/秒の場合）
- **約 10 倍の速度差**

### 8.2 楽天アフィリエイトプログラム

**基本情報:**

- **報酬率**: 商品カテゴリにより異なる（通常 1-5%程度）
- **申請**: 楽天アフィリエイトに別途登録が必要
- **審査**: 比較的簡単（ウェブサイトの公開が条件）

**メリット:**

- 申請が比較的簡単
- 報酬率が明確
- 日本語サポートが充実

### 8.3 商品データの充実度

**オフィスグッズ関連:**

- 机、椅子、キーボード、モニターなど主要商品は豊富
- 楽天市場はオフィス用品の取り扱いが多い
- 商品数は Amazon より少ないが、MVP には十分

### 8.4 データ取得の実装例

**楽天 API で取得可能な情報:**

```json
{
  "itemName": "商品名",
  "itemPrice": 価格,
  "itemUrl": "商品URL",
  "mediumImageUrls": ["画像URL"],
  "reviewAverage": 平均評価,
  "reviewCount": レビュー件数,
  "itemCode": "商品コード",
  "affiliateUrl": "アフィリエイトリンク"
}
```

### 8.5 楽天 API の実装手順（概要）

1. **楽天ウェブサービスへの登録**

   - https://webservice.rakuten.co.jp/ にアクセス
   - 無料アカウントを作成
   - アプリケーション ID を取得

2. **API の呼び出し**

   - REST API 形式
   - 認証はアプリケーション ID のみ（シンプル）
   - JSON 形式でレスポンス

3. **商品検索 API の利用**
   - `IchibaItemSearchAPI`: 商品検索
   - キーワード、カテゴリ、価格帯などで検索可能
   - ページネーション対応

## 9. 実装優先度の推奨

### 9.1 Phase 1（MVP）での実装

**最優先: 楽天 API**

1. 楽天ウェブサービスに登録
2. サンプルデータを取得（各カテゴリ 10-20 件）
3. JSON ファイルに保存
4. Next.js でプロトタイプを開発

**次点: 手動データ**

- 人気商品のみ手動で追加（Amazon 商品など）
- サンプルとして使用

### 9.2 Phase 2 での実装

1. 楽天アフィリエイトプログラムに参加
2. サービスを公開
3. ユーザーからのフィードバックを収集

### 9.3 Phase 3 での実装

1. Amazon アソシエイトプログラムに参加（実績ができた時点）
2. 手動で Amazon 商品リンクを追加
3. 十分な実績ができたら PA-API の利用を検討
